Project Overview

The Enterprise Retail Data Analytics Processing Pipeline is an automated ETL solution developed using Apache Spark (PySpark) and Python to process daily retail transaction datasets. The system performs automated data validation, cleaning, transformation, KPI generation, and optimized storage workflows to simulate production-grade enterprise data engineering pipelines.

The project demonstrates scalable batch data processing, incremental file handling, automated workflow execution using GitHub Actions, and cloud-ready analytics architecture for reporting and downstream visualization systems.

Key Features

Automated ETL Workflow

Incremental daily retail data processing using PySpark-based ETL architecture.

Dynamic File Detection

Automatically detects and processes daily uploaded retail transaction files.

Data Cleaning & Validation

Handles duplicate records, null values, invalid quantities, and inconsistent data.

KPI Analytics

Generates revenue insights, regional sales analysis, and product performance KPIs.

Spark SQL Processing

Uses Spark SQL transformations and advanced window functions for analytics workflows.

Parquet Storage

Stores optimized analytical datasets using Parquet-based storage architecture.

GitHub Actions Automation

Automated workflow execution and pipeline orchestration using GitHub Actions.

Cloud-Ready Architecture

Designed to simulate modern enterprise cloud-based data engineering systems.
